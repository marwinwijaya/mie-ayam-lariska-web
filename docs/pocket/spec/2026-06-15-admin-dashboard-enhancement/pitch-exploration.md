# Pitch Exploration: Admin Dashboard Enhancement

> Generated: 2026-06-15
> Status: Pitch Complete — Ready for Grinding

## Problem Statement

Admin dashboard saat ini hanya bisa mengelola stok (available/limited/sold_out). Admin perlu fitur lengkap untuk mengelola menu termasuk menambah/mengedit/menghapus menu, menulis deskripsi marketing, dan mengatur urutan tampilan menu yang tersync dengan Firebase.

---

## Context

**Stack:** Vanilla JS + Firebase Realtime Database + Firebase Hosting
**Architecture:** Static HTML with IIFE JavaScript modules
**Current state:** Admin can manage stock status only (available/limited/sold_out)

---

## Key Constraints

1. Static files only (no server-side functions)
2. Firebase Realtime Database (client-side SDK only)
3. Vanilla JS (no framework)
4. 24 menu items total (11 Mie Ayam + 5 Minuman + 8 Topping)

---

## Brainstorm Synthesis

### Key Insights

1. Firebase connection status — admin perlu feedback visual dan tombol reconnect manual
2. CRUD harus mengikuti pattern existing — IIFE modules dengan `window.ModuleName` pattern
3. Menu ordering perlu field `order` di Firebase — numeric field untuk sorting
4. Description field — plain text dengan limit 150-200 karakter untuk simplicity

### Ideas Worth Pursuing

1. Connection status button — toggle connected/disconnected dengan visual indicator
2. Inline editing — click pada menu item untuk edit langsung di grid
3. Drag-and-drop reordering — paling intuitif untuk ordering (HTML5 drag API)
4. Description templates — template siap pakai untuk menu types

### Discarded

- Bulk edit — overkill untuk 24 items
- History/audit log — nice-to-have, defer ke future
- HTML descriptions — increases XSS risk, plain text cukup
- Preview before save — adds complexity, inline editing sudah cukup

---

## Approach Directions

### Direction A: Inline Editing + Drag & Drop
Extend grid existing dengan inline editing dan drag-and-drop reordering.
\+ Familiar interface, cepat untuk edit
− Perlu handle drag events, complexity di mobile

### Direction B: Modal-based CRUD + Arrow Buttons
Modal popup untuk create/edit, arrow buttons untuk urutan.
\+ Clean interface, mudah di mobile
− Extra click untuk edit, less intuitive ordering

### Direction C: Hybrid — Inline Edit + Modal Create (RECOMMENDED)
Inline editing untuk edit, modal untuk create baru, drag & drop untuk urutan.
\+ Best of both worlds, flexible
− More code to maintain

**Recommended: Direction C** — Hybrid approach memberikan fleksibilitas maksimal dengan UX yang baik.
